1. 说明

  • 插件本身是一个apk包，和普通的android开发无任何区别
  • 开发的时候可以使用Android Studio进行开发，打包成apk即可

2. 插件编写

2.1. 插件目录结构


  • 插件工程本身是一个android工程
  • 插件工程中有resources文件夹，其中包括js文件夹和class.json
  • js文件夹是插件要暴露的js代码，插件被加载的时候，js文件夹中的所有文件都会被加载到执行环境中
  • class.json是用来配置暴露的java类
    • 属性值: clz: 插件类名
    • 属性值: key: 暴露给js使用的变量名 例如：
      {
   "clz": "com.android.plugin.TestPlugin1",
   "key": "p1"
}

2.2. 插件java类编写

  • java类需要一个默认构造函数
  • 如果需要使用Context，可以定义需要一个含有Context参数的构造函数

实例：

package com.android.plugin;
import android.content.Context;
public class TestPlugin1 {
   public TestPlugin1(Context context) {

   }
   public TestPlugin1() {
   }
   public String readFromPlugin() {
       return "fdsfsadfsaffasdfsadfsa";
   }
}

2.3. 插件js编写

  • 插件js代码和脚本的代码编写一样 实例:

function testPlugin() {
   logd("我是插件");
   toast("我是插件信息");
}

3. 插件使用

  • 插件编写好后，打成apk包，放在工程的plugin文件夹下，例如名称为plugin-test.apk

    3.1. 载入插件

    实例: 插件名称为plugin-test.apk
    loadECPlugin("plugin-test");

3.2. 使用插件js代码

  • 如果不包含js代码，无法调用 实例：插件名称为plugin-test.apk，包含plugin1.js文件
    //调用plugin-test插件中plugin1.js的testPlugin方法
testPlugin();

3.3. 使用插件java代码

  • 如果不包含java代码，无法调用 实例：插件名称为plugin-test.apk，包含com.android.plugin.TestPlugin1类，class.json定义的key为p1
    //调用plugin-test插件中java
p1.readFromPlugin();
